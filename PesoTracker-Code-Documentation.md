## 1. Estructura del Proyecto

La aplicación PesoTracker está organizada en los siguientes directorios principales:

```
PesoTracker/
├── Models/           # Modelos de datos
├── Services/         # Servicios para API, autenticación, etc.
├── ViewModels/       # ViewModels para la lógica de negocio
├── Views/            # Vistas SwiftUI
│   ├── Authentication/  # Vistas de autenticación
│   ├── Components/      # Componentes reutilizables
├── Utilities/        # Utilidades y extensiones
└── Resources/        # Recursos como imágenes, fuentes, etc.
```

## 6. Flujos Principales

### Flujo de Autenticación
1. La aplicación comienza en `AuthenticationContainerView`
2. El usuario puede elegir entre iniciar sesión o registrarse
3. Después de autenticarse correctamente, se muestra el dashboard

### Flujo de Registro de Peso
1. En el dashboard, el usuario pulsa "Add Weight"
2. Se muestra `AddWeightView` donde introduce los datos
3. Al guardar, se llama a `viewModel.addWeight()`
4. El dashboard se actualiza con el nuevo registro

### Flujo de Edición de Peso
1. En la tabla de pesos, el usuario pulsa "Edit" en un registro
2. Se muestra `EditWeightView` con los datos precargados
3. El usuario modifica los datos y guarda
4. Se llama a `viewModel.updateWeight()` y se actualiza el dashboard

### Flujo de Gestión de Metas
1. El usuario pulsa "Set Goal" o "Edit Goal"
2. Se muestra `GoalView` (vacío o con datos precargados)
3. El usuario establece o modifica la meta
4. Se llama a `viewModel.createGoal()` o `viewModel.updateGoal()`

## 7. Problemas Comunes y Soluciones

### Problema: Fechas incorrectas en edición
**Síntoma**: Al editar un registro, la fecha mostrada es la actual en lugar de la original.
**Solución**: 
- Asegurarse de que `setupInitialValues()` parsea correctamente la fecha del registro.
- Usar múltiples formatos de fecha para mayor compatibilidad.
- Añadir `.id(date)` al DatePicker para forzar su actualización.

```swift
// Código correcto para parsear fechas
let dateFormatter = DateFormatter()
let formats = [
    "yyyy-MM-dd'T'HH:mm:ss.SSS'Z'",
    "yyyy-MM-dd'T'HH:mm:ss'Z'",
    "yyyy-MM-dd"
]

var parsedDate: Date?
for format in formats {
    dateFormatter.dateFormat = format
    if let date = dateFormatter.date(from: weightEntry.date) {
        parsedDate = date
        break
    }
}

if let parsedDate = parsedDate {
    date = parsedDate
}
```

### Problema: Error de actor principal
**Síntoma**: Error "Main actor-isolated property can not be referenced from a nonisolated context"
**Solución**: 
- Usar `@MainActor` en funciones que acceden a propiedades aisladas al actor principal.
- Usar `await MainActor.run { }` para ejecutar código en el actor principal.
- Crear propiedades publicadas en el ViewModel para acceder desde las vistas.

```swift
// Solución para acceder a AuthenticationManager desde DashboardViewModel
@Published var username: String = ""

@MainActor
func loadUsername() {
    username = AuthenticationManager.shared.currentUser?.username ?? ""
}

// Llamada desde init()
Task {
    await MainActor.run {
        loadUsername()
    }
}
```

### Problema: ProgressView con valores fuera de rango
**Síntoma**: Error "ProgressView initialized with an out-of-bounds progress value"
**Solución**: 
- Limitar los valores de progreso entre 0 y 1.
- Especificar explícitamente el valor total como 1.0.

```swift
if let progress = viewModel.goalProgress {
    let clampedProgress = max(0.0, min(progress, 1.0))
    ProgressView(value: clampedProgress, total: 1.0)
        .frame(width: 120)
        .tint(isGoalAchieved ? .green : .blue)
}
```

### Problema: Orden incorrecto en la tabla de pesos
**Síntoma**: Los registros de peso se muestran en orden incorrecto.
**Solución**: 
- Ordenar los pesos del más antiguo al más reciente.
- Ajustar la lógica de cálculo de cambios para reflejar el orden correcto.

```swift
// Ordenar pesos del más antiguo al más reciente
self.weights = weightsResponse.data.sorted { $0.date < $1.date }

// Cálculo correcto del cambio
let current = viewModel.weights[index].weightValue
let previous = viewModel.weights[index - 1].weightValue
let change = current - previous
```
